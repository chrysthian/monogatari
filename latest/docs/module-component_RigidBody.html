Module: component/RigidBody

Component used for interaction with the box2d.js Emscripten port. Box2D has been tuned to work well with moving objects between 0.1 and 10 meters. This means objects between soup cans and buses in size should work well. Static objects may be up to 50 meters big without too much trouble. Considered units are meters, kilograms and seconds.
Creates a RigidBody.
Parameters:
type RigidBody.TYPE Type of the body: static, kinematic or dynamic
shape b2Shape Geometrical shape of the body
options Object <optional>
Properties
density number <optional>
 Material density in kg/m^2
friction number <optional>
 Material friction, usually in the range [0,1]
bounciness number <optional>
 Material bounciness, usually in the range [0,1]
angle number <optional>
 Body rotation angle in radians
isSensor boolean <optional>
 Prevents the collision to be resolved by Box2D, but retains collision information
preventTunneling boolean <optional>
 Prevents the collision to be resolved through other objects. Expensive! Use with care.
allowRotation boolean <optional>
 Prevents or allows the rotation on the body

(static, constant) FACTOR :number

Conversion factor between physics world and game world

(static) TYPE :number

Enumeration of rigid body types.
Properties:
STATIC number A body which isn’t affected by world forces, it does not react to collisions. It can’t be moved.
KINEMATIC number An hybrid body which is not affected by forces and collisions like a static body but can be moved with a linear velocity like a dynamic body.
DYNAMIC number A body which is affected by world forces and reacts to collisions.
